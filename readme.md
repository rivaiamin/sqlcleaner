# SQL Cleaner CLI 🧹

A fast, memory-efficient command-line tool written in Go to clean and sanitize large MySQL/MariaDB dump files. 

Processing massive `.sql` files with standard text tools like `sed` or `awk` can often result in memory crashes or broken syntax due to complex multi-line statements and extended `INSERT` blocks. **SQL Cleaner** solves this by streaming the file and using a smart state machine to safely strip unwanted data without loading the entire file into RAM.

## Features

- **Remove Specific Tables:** Strips both the DDL (`CREATE`, `DROP`) and the data (`INSERT`, `LOCK TABLES`) for a user-defined list of tables.
- **Remove Views:** Automatically detects and removes all View DDLs, including the temporary "dummy tables" generated by `mysqldump`.
- **Remove Stored Routines:** Removes `CREATE PROCEDURE` / `CREATE FUNCTION` blocks (handles `mysqldump` `DELIMITER` wrapping).
- **Reset Charset & Collation:** Strips `CHARACTER SET` / `CHARSET` and `COLLATE` definitions from table schemas so they inherit the target database's default settings (optional).
- **Remove GTID:** Safely removes `GTID_PURGED` statements to prevent replication errors during import.
- **Memory Safe:** Uses a custom 100MB buffer to handle massive single-line bulk inserts that normally crash standard line-by-line scanners.

## Installation

Ensure you have [Go installed](https://go.dev/doc/install) on your system. 

1. Clone the repository:
```bash
git clone [https://github.com/rivaiamin/sqlcleaner.git](https://github.com/rivaiamin/sqlcleaner.git)
cd sqlcleaner
```

2. Build the executable:
```bash
go build -o sqlcleaner main.go

```

3. (Optional) Move it to your path:
```bash
mv sqlcleaner /usr/local/bin/

```

## Usage

Run the tool by passing your input file and desired output file. Optionally exclude specific tables (`-skip-tables`) or keep only a subset (`-only-tables`; mutually exclusive with `-skip-tables`).

```bash
./sqlcleaner -in dirty_dump.sql -out clean_dump.sql -skip-tables "logs,sessions,audit_trail"
./sqlcleaner -in dirty_dump.sql -out subset.sql -only-tables "users,orders"
```

### Command Line Flags

| Flag | Description | Required | Example |
| --- | --- | --- | --- |
| `-in` | Path to the source `.sql` file. | Yes | `-in dump.sql` |
| `-out` | Path where the cleaned `.sql` file will be saved. | Yes | `-out cleaned.sql` |
| `-skip-tables` | Comma-separated list of tables to completely remove (no spaces). | No | `-skip-tables users,cache` |
| `-only-tables` | Comma-separated list of tables to keep; all other table DDL and data are removed. Cannot be combined with `-skip-tables`. | No | `-only-tables users,orders` |
| `-keep-charset` | Keep `CHARSET` / `CHARACTER SET` and `COLLATE` clauses (do not strip them). | No | `-keep-charset` |
| `-data-only` | Strip all table DDL (`CREATE TABLE`, `DROP TABLE`); keep `INSERT` and `LOCK TABLES` blocks only. Table filters still apply to data. | No | `-data-only` |

## How it Works

Instead of trying to parse the SQL file into an Abstract Syntax Tree (which is incredibly slow for multi-gigabyte files), this tool reads the file line-by-line using `bufio.Scanner`. It uses regular expressions to identify the start of unwanted statements and engages a temporary "skip mode."

Depending on what it is skipping, it ignores all subsequent lines until it detects:

- **End of statement**: default terminator is `;`
- **Stored routine end**: respects `DELIMITER <token>` (e.g. `DELIMITER ;;`) and skips until the routine closes
- **Bulk data blocks**: skips `LOCK TABLES ... WRITE` blocks until `UNLOCK TABLES`
